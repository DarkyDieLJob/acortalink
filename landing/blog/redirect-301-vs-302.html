Redirect 301 vs 302: cuándo usar cada uno

Actualizado enero 2026 · 5 min de lectura

Si estás construyendo un acortador de links, migrando un sitio, o haciendo tracking de clicks, necesitás entender la diferencia entre 301 y 302. No es un detalle técnico — afecta el SEO, el tracking y el comportamiento del navegador.

Qué es un redirect HTTP

Un redirect HTTP le dice al navegador "esta URL no está acá, andá a esta otra". El servidor responde con un código 3xx y un header Location con la URL de destino. El navegador sigue el redirect automáticamente.

301 Moved Permanently

CaracterísticaValor
SignificadoEl recurso se movió permanentemente
SEO juiceSe transfiere al destino
Google indexaLa URL de destino
Browser cacheSí — cachea el redirect
Clicks trackeablesNo — el browser cachea y no vuelve al server

Cuándo usar 301

  • Cambio de dominio (viejo.com → nuevo.com)
  • Migración HTTP → HTTPS
  • Consolidación de URLs duplicadas
  • URLs canónicas (www vs non-www)
  • Cuando querés transferir todo el SEO juice

302 Found (Moved Temporarily)

CaracterísticaValor
SignificadoEl recurso se movió temporalmente
SEO juiceNo se transfiere — se queda en la URL original
Google indexaLa URL original
Browser cacheNo — cada request pasa por el server
Clicks trackeablesSí — cada click pasa por tu server

Cuándo usar 302

  • Tracking de clicks (acortadores de links)
  • A/B testing (redirigir a variantes)
  • Redirección geográfica
  • Campañas temporales
  • Cuando querés cambiar el destino después

Por qué Acortalink usa 302

Acortalink usa 302 por tres razones:

  1. Tracking — cada click pasa por el servidor. Con 301, el browser cachea el redirect y los clicks futuros no se trackean.
  2. Editabilidad — podés cambiar el destino del link en cualquier momento. Con 301, el browser cachea y el cambio no se refleja.
  3. SEO indexable — con 302, Google indexa la URL original (tu link acortado). Si activás SEO page, Google indexa tu página con sus meta tags.

307 y 308: los primens estrictos

HTTP 1.1 agregó 307 (temporal) y 308 (permanente). La diferencia con 302/301 es que 307/308 preservan el método HTTP (GET sigue siendo GET, POST sigue siendo POST). En la práctica, para acortadores de links, 302 es suficiente porque los clicks siempre son GET.

Preguntas frecuentes

¿Cuál es la diferencia entre 301 y 302?
301 es un redirect permanente: Google transfiere el SEO juice al destino y cachea el redirect. 302 es temporal: Google no transfiere SEO juice y no cachea, por lo que cada request pasa por tu servidor.
¿Cuándo usar 301?
Usá 301 cuando el cambio es permanente: cambio de dominio, consolidación de URLs duplicadas, migración de HTTP a HTTPS, o cuando querés transferir todo el SEO juice al destino.
¿Cuándo usar 302?
Usá 302 cuando el redirect es temporal o funcional: tracking de clicks, A/B testing, redirección geográfica, o cuando querés cambiar el destino después sin afectar el SEO.
¿Qué redirect usa Acortalink?
Acortalink usa 302 por defecto. Esto permite trackear cada click (el redirect no se cachea) y cambiar el destino del link en cualquier momento sin afectar el SEO del destino.
